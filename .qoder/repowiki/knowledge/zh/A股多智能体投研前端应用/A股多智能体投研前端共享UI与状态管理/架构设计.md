- **分层结构**：采用 `services/api.ts` 作为统一HTTP客户端，`stores/` (Zustand) 管理全局业务状态（认证、分析任务），`hooks/` 封装副作用逻辑（SSE连接、断点恢复），`components/` 负责视图渲染。
- **状态驱动架构**：`analysisStore.ts` 为核心，通过 `persist` 中间件实现分析任务的本地持久化；`useAnalysisJobRecovery.ts` 在应用启动时根据持久化ID轮询后端状态并自动重连SSE，确保页面刷新后任务不中断。
- **流式通信机制**：`useSSE.ts` 基于 Fetch API 解析 Server-Sent Events，将 `agent.token`、`agent.status` 等事件实时分发至 Store，配合 `useTypeWriter.ts` 实现报告内容的打字机效果。
- **类型契约**：`types/index.ts` 定义了完整的 SSE 事件载荷、分析报告结构及业务实体，确保前后端数据交互的类型安全。
