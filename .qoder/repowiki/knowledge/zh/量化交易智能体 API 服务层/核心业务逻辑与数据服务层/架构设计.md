- **分层架构**：采用 Service-Repository 模式，`services/` 目录下的模块（如 `watchlist_service.py`, `report_service.py`）作为业务逻辑层，直接调用 `api.database` 中的 ORM 模型进行数据持久化。
- **模块化设计**：按业务领域垂直切分，包括 `auth_service`（认证与密钥管理）、`board_gold_service`（复杂的量化策略扫描引擎）、`report_service`（分析报告生命周期管理）等，各模块通过共享的 `Session` 对象协同工作。
- **依赖注入与上下文管理**：服务函数通常接收 `db: Session` 作为参数，由上层控制器或依赖注入框架管理事务边界，确保数据一致性。
- **外部集成**：`auth_service` 集成 `jwt` 和 `cryptography` 处理安全令牌；`board_gold_service` 集成 `pandas` 和 `baostock` 进行本地金融数据处理与缓存更新。
