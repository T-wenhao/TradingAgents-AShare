- 在 `board_gold_service` 中采用策略注册表（Registry）模式管理多种入场和离场策略，通过统一的 `scan` 或 `check_exit` 接口调用。
- 服务函数普遍采用显式传入 `db: Session` 和 `user_id` 的方式实现无状态数据访问，便于上层 API 路由复用。
- 在处理金融数值时，统一使用 `_to_float` 或 `_safe_float` 等辅助函数进行空值检查和类型转换，确保返回值的稳定性。
- 股票代码在不同服务间保持标准化处理，使用 `normalize_bare_symbol` 和 `normalize_display_symbol` 统一转换格式（如 600519.SH）。
