- **分层结构**：模块由三个独立的服务文件组成，分别负责持仓追踪 (`tracking_board_service`)、选股信号扫描 (`board_gold_service`) 和自选股管理 (`watchlist_service`)。
- **数据交互**：`tracking_board_service` 和 `watchlist_service` 通过 SQLAlchemy Session 与数据库交互，获取用户持仓、报告及自选列表；`board_gold_service` 则主要依赖本地 Parquet 文件缓存和 BaoStock 接口进行离线/增量数据扫描。
- **核心逻辑**：`board_gold_service` 内部实现了策略注册表模式（Registry Pattern），包含多种入场（如三阴不破阳、金凤凰）和离场策略类，并通过 `BoardGoldTaskManager` 利用线程池异步执行扫描任务及缓存更新脚本。
- **外部集成**：持仓服务集成了 `tradingagents.dataflows.interface` 获取实时行情，并关联 `ReportDB` 展示 AI 交易建议摘要。
