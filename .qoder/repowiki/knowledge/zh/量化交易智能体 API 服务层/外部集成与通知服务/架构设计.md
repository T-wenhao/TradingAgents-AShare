- 采用无状态服务层设计，各文件独立封装特定领域的业务逻辑（如 `wecom_notification_service` 处理 webhook 校验与重试，`vlm_service` 适配多模态 API）。
- 依赖倒置：服务层通过 `TYPE_CHECKING` 引用数据库模型，运行时由上层注入 `Session`，实现业务逻辑与持久层的解耦。
- 异步兼容：在同步 HTTP 请求场景（如企业微信发送）中使用 `asyncio.to_thread` 桥接异步上下文，避免阻塞事件循环。
- 配置驱动：通过环境变量（如 `TA_VLM_API_KEY`）管理敏感凭证与第三方端点，增强部署灵活性。
